# Whisper Fast GUI

Графічний інтерфейс для транскрибації аудіо та відео на основі Faster-Whisper (OpenAI Whisper).

---

## Використання на різних ОС

Скрипт підтримує Windows, Linux та macOS.

### Windows 10/11
- **Запуск:** подвійний клік по `main.py` або через ярлик (`Whisper Fast GUI.lnk`).
- **Запуск з автоматичним стартом черги:** у консолі з каталогу проекту: `python main.py --transcribe` — програма відкриється і через пів секунди автоматично почне обробку всієї поточної черги (якщо черга не порожня).
- **Без консолі:** `run_whisper.vbs`.
- **Встановлення залежностей:** `install.bat` (або кнопка [Залежності] в програмі).
- **Модель:** кнопка з назвою поточної моделі (наприклад «large-v3-turbo») — ліворуч від перемикача «Трей» — відкриває діалог вибору моделі Whisper; у діалозі є кнопка **«Завантажити модель»** для попереднього завантаження обраної моделі в пам’ять.
- **Панель / Трей / Панель + Трей:** перемикач у нижній панелі. У режимі **Трей** закриття вікна (×) не закриває програму — вона ховається в системний трей і працює далі; вихід — через меню іконки в треї. Детально — розділ «Режими Панель / Трей / Панель + Трей» нижче.
- **Автозапуск:** кнопка [Автозапуск] у програмі (поруч із перемикачем «Трей») запускає `autorun_delayed.bat` і додає програму в автозавантаження з затримкою 25 с.
- **Клік по посиланню в логу:** відкриття файлу; Shift+клік — відкрити папку та виділити файл у провіднику.
- **Звук завершення:** за відсутності файлу .mp3 у **каталозі програми** використовується системний звук Windows.

### Linux
- **Запуск:** з каталогу проекту виконати: `python3 main.py` (або `python main.py`). Для автоматичного старту обробки черги: `python3 main.py --transcribe`.
- **Drag & Drop і відображення:** потрібен робочий стіл з підтримкою X11/Wayland та tkinter.
- **Встановлення:** встановіть Python 3, FFmpeg та залежності (через pip або кнопку [Залежності]).
- **Відкриття файлів і папок з логу:** використовується `xdg-open` (пакет `xdg-utils`).
- **Звук завершення:** за відсутності .mp3 у **каталозі програми** — спроба відтворити системний звук (freedesktop); інакше звук не відтворюється.

### macOS
- **Запуск:** `python3 main.py` з каталогу проекту. Для автоматичного старту обробки черги: `python3 main.py --transcribe`.
- **Відкриття файлів з логу:** `open` (звичайний клік) та `open -R` (Shift+клік — показати у Finder).
- **Звук завершення:** як на Linux (системні звуки за відсутності .mp3 у каталозі програми).

На всіх ОС потрібні: **Python 3.9–3.13** (на практиці найстабільніше **3.11 або 3.12**), **FFmpeg** у PATH, залежності (faster-whisper, torch, pydub тощо). **Python 3.14+** часто не підходить: для нього ще немає коліс **ctranslate2** / **torch** на PyPI — встановлення падає; використовуйте 3.12. Рекомендовано відеокарту NVIDIA з CUDA для прискорення на Windows та Linux.

## Про обладнання та скрипт

Скрипт базується на технології OpenAI Whisper та бібліотеці Faster-Whisper. За замовчуванням використовується модель **large-v3-turbo** (оптимальний баланс якості та швидкості). Модель можна змінити: кнопка з назвою поточної моделі (ліворуч від перемикача «Трей») відкриває діалог вибору (tiny, base, small, medium, large-v1/v2/v3, large-v3-turbo, distil-large-v3). У діалозі відображається, які моделі вже завантажені та їхній приблизний розмір; кнопка **«Завантажити модель»** дозволяє завантажити обрану модель у пам’ять одразу, без очікування запуску транскрибації. Версія та дата застосунку відображаються у верхній частині вікна (справа від кнопок).

## Автозапуск з затримкою (Windows)

Щоб програма запускалася при вході в систему з затримкою 20–30 секунд:

**Спосіб 1 (рекомендовано):** у програмі натисніть кнопку **[Автозапуск]** (у нижній панелі після перемикача «Трей»). Відкриється вікно скрипта; після виконання ярлик з’явиться в каталозі Автозавантаження.

**Спосіб 2:** запустіть **`autorun_delayed.bat`** з папки проекту (подвійний клік або з CMD). Скрипт створить ярлик у каталозі Автозавантаження; після наступного входу в Windows програма запуститься автоматично через 25 секунд. Скрипт написаний одним рядком виклику PowerShell (без продовження рядків), щоб коректно працювати на різних ПК і при різних закінченнях рядків у файлі.

**Спосіб 3 (вручну):**
1. Відкрийте папку **Автозавантаження**: натисніть `Win+R`, введіть `shell:startup`, Enter.
2. Створіть **ярлик** на файл `start_delayed.vbs` з папки проекту і перемістіть його в папку Автозавантаження.
3. За замовчуванням затримка — **25 секунд**. Щоб змінити, відредагуйте `start_delayed.vbs`: змініть значення `delaySec = 25` на 20 або 30.

Права адміністратора не потрібні: ярлик створюється в каталозі автозавантаження поточного користувача (`%APPDATA%\...\Startup`).

Після входу в Windows програма запуститься автоматично через вказану кількість секунд (без вікна CMD).

## Режими «Панель» / «Трей» / «Панель + Трей» та робота з системним треєм

У нижній панелі програми є перемикач режиму відображення: **Панель**, **Трей**, **Панель + Трей**. Від нього залежить, де з’являється програма (панель задач, область сповіщень біля годинника, чи обидва) і **що відбувається при натисканні кнопки закриття вікна (×)**.

Потрібні бібліотеки **pystray** та **Pillow** (кнопки [Залежності] або [Оновлення]). Якщо вони не встановлені, режими «Трей» та «Панель + Трей» недоступні (у лозі з’явиться попередження).

### Режим «Панель»
- Відображається лише на **панелі задач**; іконки в системному треї немає.
- **Закриття вікна (×):** з’являється діалог «Закрити програму?» — підтвердження закриває застосунок.

### Режим «Трей»
- Програма показується лише як **іконка в системному треї** (область сповіщень біля годинника). Вікно при запуску приховане; подвійний клік по іконці або пункт меню **«Показати вікно»** відкривають вікно.
- **Закриття вікна (×):** вікно **не закривається** — воно ховається в трей, програма продовжує працювати у фоні (зокрема може працювати черга та слідкування за каталогом).
- **Повне закриття програми:** у контекстному меню іконки в треї виберіть **«Вихід»** — з’явиться звичайний діалог підтвердження закриття.

### Режим «Панель + Трей»
- Програма одночасно на **панелі задач** і в **системному треї**.
- **Закриття вікна (×):** як у режимі «Панель» — з’являється діалог закриття; підтвердження закриває застосунок.
- Якщо потрібно лишити програму працювати у фоні без закриття — згорніть вікно кнопкою «−» (мінімізація), а не закривайте через ×.

### Підсумок
| Режим           | × закриває вікно? | Де закрити програму повністю?        |
|-----------------|-------------------|--------------------------------------|
| Панель          | Так (через діалог)| Кнопка × → «Так» у діалозі           |
| Трей            | Ні (ховає в трей) | Меню іконки в треї → «Вихід» → «Так» |
| Панель + Трей   | Так (через діалог)| Кнопка × → «Так» у діалозі           |

## Черга (таблиця)

Підпис блоку — **«Черга»** (або «Queue» / «Очередь» залежно від мови). При наведенні курсора на цей підпис з’являється підказка з поясненням роботи таблиці та відрізків часу.

Черга відображається як **таблиця** з колонками:

| Колонка           | Опис |
|------------------|------|
| **#**            | Порядковий номер. |
| **Назва файлу**  | Ім'я файлу (плюс «– оброблено», якщо вже оброблений). |
| **Початок**      | Час початку обробки (за замовчуванням 00:00:00,000). |
| **Кінець відр. 1** | Кінець першого відрізка (за замовчуванням порожньо). |
| **Кінець відр. 2** | Кінець другого відрізка (за замовчуванням порожньо). |
| **Кінець**       | Час кінця обробки (за замовчуванням — тривалість файлу). |

- При додаванні файлу **Початок** = 00:00:00,000, **Кінець** = тривалість файлу (з ffprobe/pydub).
- **Подвійний клік** по рядку — діалог редагування діапазону часу (Початок, Кінець відр. 1/2, Кінець).
- Черга **зберігається** в `request_queue.json` (додавання, очищення, перетягування, редагування).
- При **запуску** програми черга підвантажується з `request_queue.json` (файли, яких уже немає на диску, пропускаються).

Обробка виконується лише в діапазоні **[Початок — Кінець]** для кожного рядка. Так можна обробити один файл частинами (наприклад 0–20 хв англійською, 20–60 хв російською).

## Функціонал

Перетворює мовлення з аудіо та відео на текст. Вихідні файли створюються у каталозі збереження або поруч із вихідним файлом:

1. **.txt** — чистий текст транскрибації (кожен сегмент з нового рядка).
2. **.srt** — субтитри: нумерація сегментів, часові мітки, текст.

Опційно (при увімкненій опції «Зберегти витягнуте аудіо (MP3)»):
3. **_audio.mp3** — витягнута аудіодоріжка (для повного файлу — один файл; для відрізка — окремий файл з суфіксом часу).

**Обробка по діапазону (відрізками):** якщо в таблиці задано Початок і/або Кінець відмінні від 0 і тривалості файлу, обробляється лише цей відрізок. Результати зберігаються в **окремі файли** з суфіксом часу, наприклад:
- `ім'я_00-00-00_00-20-00.txt` / `.srt` — відрізок 0:00–0:20;
- `ім'я_00-20-00_01-00-00.txt` / `.srt` — відрізок 0:20–1:00.
При увімкненому «Зберегти Mp3» для кожного відрізка створюється окремий `ім'я_..._audio.mp3` за тим самим діапазоном.

## Режими обробки

- **Один файл у черзі:** виберіть файл і натисніть [Старт] — діалог не показується.
- **Кілька файлів:** при натисканні [Старт] з’являється діалог: «Тільки обраний файл», «Усі файли в черзі» або «Скасувати».
- **Тільки нові:** обробляються лише файли без позначки «– оброблено».
- **Вся черга:** обробка всіх файлів по порядку.

## Налаштування пристрою та моделі

- **AUTO** — автоматичний вибір (відеокарта NVIDIA за наявності, інакше CPU).
- **GPU** — примусово відеокарта (якщо CUDA доступна). Для старих карт — безпечний режим int8.
- **CPU** — примусово процесор, int8 (підходить також для AMD Radeon та інших GPU без CUDA).

**Вибір моделі:** кнопка з назвою поточної моделі (наприклад «large-v3-turbo») відкриває діалог, де можна обрати модель зі списку (tiny, base, small, medium, large-v1/v2/v3, large-v3-turbo, distil-large-v3), переглянути, які з них уже завантажені та їхній розмір, і натиснути **«Завантажити модель»**, щоб одразу завантажити обрану модель у пам’ять (без очікування старту транскрибації). Модель завантажується один раз і використовується для всіх файлів (Singleton). Завантажені моделі зберігаються в кеші Hugging Face Hub (шлях показано в підказці до кнопки та в діалозі).

**Додаткові опції:**
- Відтворити звук по завершенні черги.
- Зберегти витягнуте аудіо (MP3) — для повного файлу один `<ім'я>_audio.mp3`; для відрізка — окремий файл з суфіксом часу (наприклад `<ім'я>_00-20-00_01-00-00_audio.mp3`).

| [Каталог збереження] | Каталог для вихідних файлів (порожнє поле — збереження поруч із вихідним) |
| [Довідка]            | Ця довідка |

**Прапорці:** «Відтворити звук по завершенні черги», «Зберегти витягнуте аудіо (MP3)».

**Слідкування за каталогом:** при увімкненій позначці програма слідкує за вказаним каталогом. Кожен **новий** підтримуваний файл:
1. **Додається в чергу** (таблицю) з діапазоном 00:00:00 — тривалість файлу.
2. **Автоматично запускається обробка** саме цього файлу (режим «тільки обраний»). Якщо в цей момент уже виконується інша обробка, нова не стартує (у лог виводиться попередження); файл лишається в черзі — обробити його можна натиснувши [Старт] після завершення поточної задачі. Одночасно виконується лише **одна** задача транскрибації.

## Поведінка клавіш

- **Enter** — якщо черга порожня: відкривається діалог «Додати файли»; якщо є файли: запуск транскрибації (як кнопка [Старт]).
- **Пробіл (Space)** — перемикає позначку «Зберегти Mp3» (у полях вводу працює як звичайний пробіл).
- **Ctrl+V** — вставка шляху з буфера обміну в поле «Слідкування за каталогом».

## Корисні можливості

- **Таблиця черги:** підпис «Черга»; при наведенні на нього — підказка з поясненням колонок та відрізків часу. Шість колонок (№, назва, початок, кінець відр. 1/2, кінець); подвійний клік — редагування часу.
- **Drag & Drop:** перетягування файлів у вікно; перетягування рядків у таблиці змінює порядок черги.
- **Збереження черги:** `request_queue.json` — черга зберігається при змінах і підвантажується при запуску.
- **Розумна черга:** оброблені файли позначаються «– оброблено».
- **Посилання в логу:** клік — відкрити файл; Shift+клік — відкрити папку з файлом у провіднику.
- **Контекстне меню логу:** правий клік — копіювання тексту.
- **Розмір логу:** автоматично обмежується (старі рядки видаляються зверху), щоб уникнути надмірного споживання пам’яті при довгих сесіях.
- **Відлагодження:** якщо перед запуском встановити змінну середовища `DEBUG=1`, у лог виводиться повний traceback при помилках транскрибації.
- **Прогрес-бар** та часові мітки сегментів під час обробки.
- **Фільтр VAD:** Voice Activity Detection увімкнено за замовчуванням.
- **Іконка:** вікно та панель задач використовують `favicon.ico` з папки проекту; для іконки в треї при відсутності файлу використовується резервна сіра іконка.
- **Діалоги:** усі спливаючі вікна (вибір моделі, редагування часу рядка, вибір «тільки обраний / усі файли», довідка) відображаються по центру головного вікна програми (або екрана).

## Встановлення та налаштування

### Первинне встановлення

1. **Встановіть Python 3.9+** (рекомендовано 3.10+). При встановленні позначте «Add Python to PATH».  
   https://www.python.org/downloads/

2. **Встановіть FFmpeg** і додайте його в PATH.  
   https://ffmpeg.org/download.html

3. **Запустіть `install.bat`.** Скрипт:
   - перевіряє Python та вже встановлені пакети;
   - оновлює pip, setuptools, wheel;
   - встановлює PyTorch (CUDA 12.1), faster-whisper, ctranslate2;
   - пропускає nvidia-cublas-cu12 та nvidia-cudnn-cu12 при першому встановленні (їх можна встановити з GUI: [Оновлення] або [Залежності]);
   - встановлює pygame, pydub, tkinterdnd2-universal, pystray, Pillow;
   - перевіряє пакети, FFmpeg та CUDA.

4. **Запуск:**
   - `run_whisper.vbs` — подвійний клік (вікно консолі не показується).
   - Ярлик можна створити вручну: ціль — `run_whisper.vbs` або `main.py`.  
   Для відлагодження з CMD: `py main.py` (з папки проекту).

### Автоматичне встановлення

При першому запуску скрипт перевіряє та встановлює відсутні залежності (без компонентів NVIDIA). Бібліотеки NVIDIA встановлюються лише через [Оновлення] або [Залежності] в GUI.

### Встановлення вручну

У командному рядку в каталозі проекту виконайте: `python installer.py`.

## Залежності (таблиця)

Усі пакети, потрібні для роботи програми, та призначення кожного:

| Пакет | Призначення |
|-------|-------------|
| **pip** | Встановлення та оновлення пакетів Python. |
| **setuptools** | Збірка та встановлення пакетів. |
| **wheel** | Формат бінарних пакетів для швидкого встановлення. |
| **torch** | PyTorch — основа для роботи моделі Whisper на CPU або GPU. |
| **torchvision** | Додаткові компоненти PyTorch (потрібні разом із torch). |
| **torchaudio** | Робота з аудіо в PyTorch. |
| **faster-whisper** | Модель розпізнавання мовлення (обгортка над Whisper). |
| **ctranslate2** | Швидке інференс-ядро для моделей перетворення (використовується faster-whisper). |
| **nvidia-cublas-cu12** | Бібліотеки NVIDIA CUDA 12 для прискорення на GPU (встановлюються з GUI). |
| **nvidia-cudnn-cu12** | Бібліотеки cuDNN для NVIDIA GPU (встановлюються з GUI). |
| **pydub** | Обробка аудіо: витягування відрізків, експорт у MP3, отримання тривалості. |
| **pygame** | Відтворення звуку завершення обробки (звукове сповіщення). |
| **tkinterdnd2-universal** | Перетягування файлів (Drag & Drop) у вікно та всередині таблиці черги. |
| **pystray** | Іконка програми в системному треї (область сповіщень). |
| **Pillow** | Завантаження та підготовка зображення іконки для трею (favicon.ico). |
| **pyaudioop** | Заміна видаленого модуля `audioop` у Python 3.13+ (потрібен для pydub). |
| **ffmpeg** | Системна програма (не pip): декодування аудіо/відео, перевірка тривалості. Потрібно встановити окремо і додати в PATH. |

**Примітки:**
- **nvidia-cublas-cu12** та **nvidia-cudnn-cu12** при першому запуску (install.bat або автоустановка) не ставляться; їх встановлюють кнопки [Оновлення] або [Залежності] в програмі.
- **pyaudioop** потрібен лише для Python 3.13+; за потреби встановлюється автоматично.
- **tkinter** — частина стандартної поставки Python; використовується для графічного інтерфейсу.

## Безпека та стабільність

- **Безпечне завершення:** при закритті застосунку модель вивантажується з пам’яті, очищується кеш CUDA.
- **Скасування операції:** миттєва зупинка з коректним завершенням поточного сегмента.
- **Обробка помилок:** логування та коректна реакція на збої. Якщо під час обробки виникає помилка типу «list index out of range», програма показує підказку: у файлі може **відсутня звукова доріжка** або звуку немає — перевірте файл (наприклад, відкрийте його в медіаплеєрі).
- **Одна задача одночасно:** у будь-який момент виконується лише одна транскрибація; повторний [Старт] або новий файл зі слідкування під час виконання не запускають другу задачу паралельно.
